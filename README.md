# DriverPass System Design Portfolio

## Project Overview: DriverPass
**Client:** DriverPass Driving School  
**System Type:** Cloud-based Web Application  

The **DriverPass** project involved designing a modern, centralized platform to automate the operations of a local driving school. The primary goal was to improve student preparation for the DMV exam while reducing the administrative burden of manual scheduling. Key features include an automated driving lesson booking system, secure third-party payment processing, and interactive practice exams.

---

## Reflection

### **Strengths & Areas for Improvement**
* **What I did well:** I excelled at visual modeling, translating high-level business needs into technical blueprints through UML Class, Sequence, and Activity diagrams. These models ensure a shared understanding between stakeholders and developers.
* **Proposed Revision:** If I were to revise this work, I would expand the **IT Admin's dashboard**. Specifically, I would add automated reporting tools for management to track long-term revenue trends and instructor performance, providing more business intelligence beyond basic scheduling.

### **Interpreting User Needs**
I interpreted the client’s needs by identifying core "pain points," such as the inefficiency of phone-based booking and the risk of double-booking. I addressed these by implementing automated availability checks and role-based access. 

**Why user needs matter:** Considering the user’s needs is vital because a system’s success depends on its usability and how effectively it solves the human problems it was built to address. A technically perfect system is a failure if it doesn't align with how the client works.

### **Software Design Approach**
My design philosophy is **iterative and visual**. I define system boundaries first, then drill down into specific logic flows and data structures. In future projects, I will continue to use Lucid for rapid prototyping, as these tools allow for quick adjustments during the requirements-gathering phase, ensuring stakeholders and technical teams are in alignment before development begins.
